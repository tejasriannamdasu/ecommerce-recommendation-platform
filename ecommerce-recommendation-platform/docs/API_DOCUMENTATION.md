# API Documentation

Base URL: `http://localhost:8000/api/v1`
Interactive docs (auto-generated by FastAPI): `http://localhost:8000/docs`

Auth: JWT Bearer token. Obtain via `/auth/login`, then send
`Authorization: Bearer <token>` on protected routes.

## Authentication

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/auth/register` | – | Create a new account |
| POST | `/auth/login` | – | Returns `{ access_token, token_type }` |
| POST | `/auth/logout` | Required | Client discards token (stateless JWT) |
| GET | `/auth/me` | Required | Current user profile |

## Products

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| GET | `/products` | – | List/filter products (`category_id`, `min_price`, `max_price`, `brand`, `sort_by`) |
| GET | `/products/search?q=...&semantic=true` | – | Keyword or semantic (FAISS) search |
| GET | `/products/autocomplete?prefix=...` | – | Search-as-you-type suggestions |
| GET | `/products/{id}` | – | Product detail |
| POST | `/products/{id}/view` | Required | Logs a view (feeds recommendations) |
| GET | `/products/{id}/similar` | – | Content-based similar products |

## Recommendations

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| GET | `/recommendations/personalized` | Required | Hybrid recommendations for the logged-in user |
| GET | `/recommendations/trending` | – | Popularity-ranked (cold-start / anonymous) |

Response shape (`RecommendationResponse`):
```json
{
  "user_id": 12,
  "strategy": "hybrid",
  "items": [
    {
      "product": { "id": 45, "title": "...", "price": 59.99, "...": "..." },
      "score": 0.87,
      "reason": "Matches your taste profile and users with similar behavior also engaged with it"
    }
  ]
}
```

## Wishlist / Profile

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| POST | `/wishlist/{product_id}` | Required | Add to wishlist |
| DELETE | `/wishlist/{product_id}` | Required | Remove from wishlist |
| GET | `/wishlist` | Required | List wishlist items |
| GET | `/recently-viewed` | Required | Recently viewed products |
| GET | `/profile` | Required | Current user profile |

## Analytics (admin only)

| Method | Endpoint | Auth | Description |
|---|---|---|---|
| GET | `/analytics/dashboard` | Admin role | Most viewed/purchased, top categories, user activity |

## Rate limiting

All endpoints are rate-limited by client IP (default: 100 requests/minute,
configurable via `RATE_LIMIT_DEFAULT` in `.env`). Exceeding the limit
returns HTTP 429.
